# `.env` 自動読込 仕様

## スコープ
- プロジェクト直下の `.env` を起動時に自動読込する。
- `config.py` の機密設定を環境変数経由で参照できるようにする。
- 未設定時の起動メッセージを `.env` / 環境変数前提の案内に更新する。

## 制約
- 既存の `python-dotenv` 依存関係を利用する。
- 実トークンや実認証情報はソースコードへ直書きしない。
- 既存の起動方法 (`python bot.py`, `start.bat`) は維持する。

## 受け入れ条件
- `.env` に `DISCORD_TOKEN` を定義すれば、追加の環境変数設定なしで `config.py` から参照できる。
- `.env` に `PAYPAL_EMAIL` と `PAYPAL_PASSWORD` を定義すれば、`config.py` の設定へ反映される。
- `.env` が存在しない場合でもクラッシュせず、既存のデフォルト値で動作する。
- `DISCORD_TOKEN` 未設定時のエラーメッセージが `.env` / 環境変数を案内する。

## 対象外
- すべての設定項目を `.env` 化すること。
- Bot の挙動変更や機能追加。
