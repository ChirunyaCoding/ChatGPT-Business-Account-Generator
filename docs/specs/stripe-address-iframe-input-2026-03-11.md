# Stripe住所入力のiframe対応強化仕様（2026-03-11）

## スコープ
- PayPal選択後のStripe住所入力で、Shadow DOMやiframe内のフィールド検出を強化する。
- `addressFrame.evaluate()` による深いDOM探索を追加する。
- 住所フィールドのセレクタを拡充する。
- 住所フォームを持つStripe iframeを、実フィールドの存在で判定する。
- フランス請求先データ生成をユーティリティ化し、固定名を廃止する。
- 回帰テストを追加する。

## 制約
- 既存の自動化フローの意図と順序は変更しない。
- 依存関係は追加しない。
- 秘密情報は `.env` のみを使用し、コードへハードコードしない。

## 受け入れ条件
- 住所入力が `addressFrame.evaluate()` で最低3項目以上入力できる。
- 住所用iframeが `iframe[name*="StripeFrame"]` の先頭一致ではなく、住所フィールドの実在で選ばれる。
- 生成される請求先名が固定値ではなく、フランス向けプロフィールとして構築される。
- 新しいセレクタ群がユーティリティ経由で参照される。
- 追加した回帰テストが成功する。

## 対象外
- Stripe決済フローの大規模改修。
- Puppeteer全体の設計変更。
